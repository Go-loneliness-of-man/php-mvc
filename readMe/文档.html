路由

路径：/config/route.php

默认路由与 ThinkPHP 相同：http://域名/模块/控制器/操作

支持路由注册，支持 params 传参，路由注册写法如下：

$route = [ ['路由路径', '模块名', '控制器名', '操作名'], ['路由路径/:参数 1 名/:参数 2 名/:参数 3 名', '模块名', '控制器名', '操作名'] ];

配置文件

支持全局配置文件、模块配置文件。

全局配置文件：

路径：/config/config.php

作用于整个应用。

模块配置文件：

路径：/app/模块名/config.php

用于为模块进行个性化配置，该文件会覆盖全局配置部分参数。格式固定：

<?php $模块名 = [ // 配置变量名与模块名保持一致 ];

公共函数

支持全局公共函数、模块公共函数，需在 config 文件（全局、模块都可以）内配置，将文件名加入 'publicFuncFile' 即可。

全局公共函数：

路径：/publicFunc/文件名

作用于整个应用。

模块公共函数：

路径：/app/模块名/文件名

只在当前模块内生效，若与全局公共函数文件同名，会覆盖掉全局公共函数文件。

中间件

路径：/middleware/文件名

中间件是个类文件，可以有很多个中间件。中间件必须在 config 文件（全局、模块都可以）的 'middleware' 中进行配置才能生效，写法如下：

'middleware' => [ ['中间件名', '被执行的方法名'] ]

中间件的执行顺序与在数组中的顺序相同，中间件不会被实例化，所以其方法必须是静态方法，被默认传入 $_REQUEST 作为参数。
